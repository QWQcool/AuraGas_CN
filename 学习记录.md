### 模块依赖关系分析（AuraGas.Build.cs）
```csharp
PublicDependencyModuleNames.AddRange(new string[] { 
    "Core", "CoreUObject", "Engine",     // UE核心模块
    "InputCore", "EnhancedInput",         // 输入系统
    "GameplayAbilities"                   // GAS系统
});

PrivateDependencyModuleNames.AddRange(new string[] { 
    "GameplayTags", "GameplayTasks"       // GAS辅助模块
});
```

**关键理解**：
- **PublicDependency**：其他模块可以访问的依赖
- **PrivateDependency**：仅本模块内部使用的依赖
- **模块化设计**：每个功能独立成模块，便于维护和复用

## C++在UE5中的特殊用法详解

### 1. UE反射系统与宏定义
```cpp
// UCLASS宏 - 使C++类在UE编辑器和蓝图中可用
UCLASS()
class AAuraCharacterBase : public ACharacter, public IAbilitySystemInterface
{
    GENERATED_BODY()  // 必须的反射代码生成宏
    
public:
    // UPROPERTY宏 - 属性序列化和蓝图暴露
    UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
    TObjectPtr<UAbilitySystemComponent> AbilitySystemComponent;
    
    // UFUNCTION宏 - 函数蓝图调用支持
    UFUNCTION(BlueprintCallable)
    virtual void InitializeAttributes();
};
```

### 2. 智能指针与内存管理
```cpp
// TObjectPtr - UE5新型智能指针（替代裸指针）
TObjectPtr<UAbilitySystemComponent> AbilitySystemComponent;

// 与传统C++智能指针对比：
// std::shared_ptr - 引用计数，通用C++
// TSharedPtr - UE的共享指针，线程安全
// TObjectPtr - 专门用于UObject派生类，支持垃圾回收
```

### 3. 委托系统（Delegates）
```cpp
// 多播委托 - 一对多的事件通知
DECLARE_MULTICAST_DELEGATE_OneParam(FOnHealthChanged, float);
FOnHealthChanged OnHealthChanged;

// 使用示例：
OnHealthChanged.Broadcast(NewHealthValue);  // 广播事件
OnHealthChanged.AddUObject(this, &AWidget::UpdateHealthBar);  // 绑定回调
```

### 4. 网络复制系统
```cpp
// 网络复制属性
UPROPERTY(ReplicatedUsing = OnRep_Health)
float Health;

// 复制回调函数
UFUNCTION()
void OnRep_Health();

// 必须实现的网络复制注册
virtual void GetLifetimeReplicatedProps(TArray<FLifetimeProperty>& OutLifetimeProps) const override;
```

## 已实现功能模块深度分析

### 1. 角色移动系统（Enhanced Input）
**技术演进对比**：
- **传统InputSystem**：基于按键映射，硬编码
- **Enhanced Input**：基于输入动作，支持复杂输入处理

```cpp
// Enhanced Input配置流程：
// 1. 创建Input Mapping Context
// 2. 定义Input Actions（移动、跳跃等）
// 3. 在PlayerController中绑定
void AAuraPlayerController::SetupInputComponent()
{
    // 现代输入处理方式
    if (UEnhancedInputComponent* EnhancedInputComponent = CastChecked<UEnhancedInputComponent>(InputComponent))
    {
        EnhancedInputComponent->BindAction(MoveAction, ETriggerEvent::Triggered, this, &AAuraPlayerController::Move);
    }
}
```

### 2. GAS系统架构深度解析
**Gameplay Ability System核心概念**：
- **AttributeSet**：属性定义和管理
- **AbilitySystemComponent**：能力执行组件
- **GameplayEffect**：属性修改效果
- **GameplayAbility**：可执行的能力

```cpp
// 属性访问器宏简化代码
#define ATTRIBUTE_ACCESSORS(ClassName, PropertyName) \
    GAMEPLAYATTRIBUTE_PROPERTY_GETTER(ClassName, PropertyName) \
    GAMEPLAYATTRIBUTE_VALUE_GETTER(PropertyName) \
    GAMEPLAYATTRIBUTE_VALUE_SETTER(PropertyName) \
    GAMEPLAYATTRIBUTE_VALUE_INITTER(PropertyName)

// 使用示例：
ATTRIBUTE_ACCESSORS(UAuraAttributeSet, Health)
```

### 3. UI系统MVC模式实现
**现代UI架构**：
```
Model（数据） → Controller（逻辑） → View（显示）
   AttributeSet     WidgetController        UserWidget
```

**设计优势**：
- **数据与显示分离**：属性变更不影响UI布局
- **逻辑集中管理**：所有UI逻辑在Controller中处理
- **易于测试**：可以单独测试每个组件

## 常见问题与解决方案

### 1. 编译错误：无法找到头文件
**问题**：模块依赖配置错误
**解决**：检查Build.cs文件中的依赖模块名称

### 2. 蓝图无法调用C++函数
**问题**：缺少UFUNCTION宏或参数类型不匹配
**解决**：确保使用正确的宏和蓝图兼容的参数类型

### 3. 网络同步问题
**问题**：客户端看不到服务器变化
**解决**：检查Replicated属性和GetLifetimeReplicatedProps实现

### 4. 内存泄漏
**问题**：UObject没有被正确垃圾回收
**解决**：使用TObjectPtr，避免手动new/delete

## 已实现功能模块

### 1. 角色移动系统
- **实现方式**: 使用UE5的Enhanced Input系统
- **控制方式**: WASD键盘输入
- **核心文件**: 
  - `AuraPlayerController.cpp` - 移动逻辑实现
  - `AuraCharacterBase.h` - 角色基类定义
- **技术特点**: 
  - 支持网络同步（`bReplicates = true`）
  - 基于控制器的旋转方向移动
  - 前后左右移动独立控制

### 2. 鼠标高光检测系统
- **实现方式**: 射线检测（CursorTrace）
- **检测逻辑**: 每帧执行鼠标光标下的Actor检测
- **状态管理**: 
  - LastActor/ThisActor记录前后帧的检测结果
  - 五种状态处理（A-E）确保高光切换流畅
- **接口设计**: `IEnemyInterface`接口定义高光方法

### 3. 敌人交互系统
- **接口定义**: `EnemyInterface.h`
- **核心方法**: 
  - `HighlightActor()` - 高光显示
  - `UnHighlightActor()` - 取消高光
- **应用场景**: 鼠标悬停敌人时显示高光效果

### 4. 属性管理系统
- **架构设计**: Gameplay Ability System (GAS)
- **属性定义**: 
  - 血量（Health/MaxHealth）
  - 法力（Mana/MaxMana）
- **技术实现**: 
  - `AuraAttributeSet.h` - 属性集定义
  - 支持网络复制和属性变更回调
  - 使用宏定义简化属性访问器

### 5. UI界面系统
- **HUD系统**: `AuraHUD` - 主界面管理
- **Widget系统**: 
  - `WBP_Overlay` - 覆盖层界面
  - `WBP_HealthGlobe` - 血条显示
  - `WBP_ManaGlobe` - 法力条显示
- **控制器**: `OverlayWidgetController` - UI逻辑控制

### 6. 药瓶碰撞系统
- **实现方式**: `AuraEffectActor`
- **碰撞检测**: Sphere组件重叠检测
- **事件处理**: 
  - `OnOverlap()` - 开始重叠
  - `EndOverlap()` - 结束重叠
- **功能**: 角色与药瓶碰撞时增加血量
